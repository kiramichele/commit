IPO in Python

Python 3 ~20 min Standalone — follows Lesson 0.5
What you're doing

Every program does the same thing: take input, process it, produce output. You're going to write three programs that make that pattern concrete — one at a time. The focus isn't on memorizing syntax. It's on recognizing that the same IPO structure appears in programs of every size and complexity.

Program 1 of 3
Program 1
Name and age
Starter program
Input: name + age Process: add 10 Output: sentence

Ask the user for their name and current age. Calculate what their age will be in 10 years. Print a sentence using both values.

program1.py
# Program 1 — Name and age # INPUT: ask the user for two pieces of information name = input("What is your name? ") age = int(input("How old are you? ")) # PROCESS: calculate age in 10 years future_age = _______________ # add 10 to age # OUTPUT: print a sentence using both values print(_______________) # e.g. "In 10 years, Kira will be 35."
Expected output (example values)
What is your name? Kira How old are you? 25 In 10 years, Kira will be 35.
Why int()?

input() always gives you a string — even if the user typed a number. "25" and 25 are different things in Python: you can't add 10 to "25" (try it and see what error you get). Wrapping input() in int() converts the string to an integer so you can do math with it.

Program 2
Temperature converter
A little harder
Input: °F Process: convert formula Output: °C

Ask the user for a temperature in Fahrenheit. Convert it to Celsius using the formula (°F − 32) × 5/9. Print the result rounded to one decimal place.

program2.py
# Program 2 — Temperature converter # INPUT: get temperature from the user fahrenheit = float(input("Enter temperature in Fahrenheit: ")) # PROCESS: apply the conversion formula (°F - 32) × 5/9 celsius = _______________ # write the formula here # PROCESS: round to 1 decimal place celsius_rounded = round(_______, 1) # what goes in the first blank? # OUTPUT: print the result print(f"{fahrenheit}°F is {_______________}°C")
Expected output (three test values)
Enter temperature in Fahrenheit: 32 32°F is 0.0°C Enter temperature in Fahrenheit: 212 212°F is 100.0°C Enter temperature in Fahrenheit: 98.6 98.6°F is 37.0°C
Test with known values

Good programmers test their code with values they already know the answer to. 32°F should give 0°C (water freezes). 212°F should give 100°C (water boils). 98.6°F should give 37°C (body temperature). If your formula is wrong, these tests will catch it immediately.

Program 3
Arithmetic all at once
Most output
Input: two numbers Process: four operations Output: four results

Ask the user for two numbers. Calculate and print their sum, difference, product, and quotient — each on its own labeled line. Handle division carefully: what happens if someone enters 0 as the second number?

program3.py
# Program 3 — Arithmetic calculator # INPUT: two numbers from the user a = float(input("Enter first number: ")) b = float(input("Enter second number: ")) # PROCESS + OUTPUT: calculate and print each operation print(f"Sum: {_______________}") # a + b print(f"Difference: {_______________}") # a - b print(f"Product: {_______________}") # a * b # Division needs a safety check — what if b is 0? if _______________: # check: is b equal to zero? print("Quotient: cannot divide by zero") else: print(f"Quotient: {_______________}") # a / b
Expected output (two examples)
Enter first number: 12 Enter second number: 4 Sum: 16.0 Difference: 8.0 Product: 48.0 Quotient: 3.0 Enter first number: 7 Enter second number: 0 Sum: 7.0 Difference: 7.0 Product: 0.0 Quotient: cannot divide by zero
What happens without the check?

Try removing the if block and dividing by zero anyway. Python raises a ZeroDivisionError and the program crashes. This is your first example of defensive programming — anticipating what users might do wrong and handling it gracefully instead of crashing.

Extension Round the results — no trailing decimals ▼

Right now the output shows 16.0, 8.0, etc. — because float always includes a decimal point. Modify the output so it prints whole numbers without decimals when the result is a whole number (e.g. 16 not 16.0), but keeps decimals when they're needed (e.g. 3.5).

Hint: look up Python's :.6g format specifier inside an f-string — it automatically drops unnecessary trailing zeros. f"{value:.6g}" will print 16 for 16.0 and 3.5 for 3.5.

The pattern

What you just built — three times

All three programs follow the same structure. The complexity changed — Program 3 has a conditional that 1 and 2 don't — but the shape is identical every time:

Input
input()
Get data from the user
Process
Math, logic, decisions
Transform the data
Output
print()
Show the result

The tip calculator you'll eventually build for Exercise 3.1 in Unit 3 is a more polished version of Program 1. The password strength checker is a more complex version of Program 3's conditional. The pattern scales — every program you write this year will be IPO at its core.
