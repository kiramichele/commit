Privacy policy decoder

Python 3 ~25 min Standalone — follows Lesson 0.4
What you're doing

You've seen what data apps collect. Now you're going to read — and parse — the legal document where they admit it. You'll write Python code that reads a privacy policy excerpt, counts how many times suspicious words appear, flags every sentence that mentions selling data, and prints a plain-English summary. String manipulation with a real-world purpose.

Read it first

Scroll through the policy you chose before you start coding. Pick out two or three sentences that surprise you. Your code is going to find patterns — but your brain catches meaning. Both matter.

Step 2 — your tasks

Three things your program needs to do

Work through these in order. Each part builds on the last. The starter code below is already split into sections — fill in one blank at a time.

1
Count how many times red-flag words appear

Your red-flag words are: "share", "sell", "third party", "partner", "advertis", "transfer", "collect". For each word, count how many times it appears in the full policy text (case-insensitive). Print the count for each word.

2
Flag every sentence that mentions selling or sharing data

Split the policy into individual sentences. Loop through them. If a sentence contains "sell", "sold", or "share", print it with a [FLAG] label in front. This is the most important output — these are the sentences lawyers spend careers writing carefully.

3
Print a plain-English summary

Using the counts from Part 1, print a two-line summary: how many times the policy mentions sharing data, and how many times it mentions selling data. Format it so a non-technical person could read it and understand what they're signing up for.

The key method: .count()

Python strings have a built-in .count(substring) method that returns how many times a substring appears. "hello world".count("l") returns 3. You're using this on a very long string — the entire privacy policy — to find patterns a human reader would miss.

Common mistake

If your flag count is 0, check two things: (1) did you call .lower() on the sentence before checking? "Share" in sentence won't match "share". (2) Is your or condition using in correctly? The pattern is "word" in sentence_lower, not "word" in "sentence_lower" (with quotes around the variable name).

Extension Find the most suspicious sentence ▼

Write a function called suspicion_score(sentence) that takes a single sentence and returns how many red-flag words it contains. Then loop through all sentences, find the one with the highest score, and print it with a label: [MOST CONCERNING].

This is exactly how automated content moderation and legal review tools work — not by reading every document, but by scoring them and surfacing the highest-risk passages for human review.
