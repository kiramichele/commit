Pseudocode translator

Python 3 + AP Pseudocode ~25 min Standalone — follows Lesson 0.5
What you're doing

The AP exam uses its own pseudocode notation — not Python, not JavaScript. Students who see it for the first time in April panic. You're seeing it in week two so it feels familiar by exam day. You'll translate in both directions: AP pseudocode → Python, and Python → AP pseudocode. The harder direction is Python → pseudocode, because it forces you to think about what your code is doing, not just how it's written.

The one thing to remember going in

AP pseudocode lists are 1-indexed — the first item is at position 1. Python lists are 0-indexed — the first item is at position 0. When you see list[1] in pseudocode it means the first element. In Python, list[1] means the second element. Watch for this in Part 2.

Part 1 — AP pseudocode → Python

Read the pseudocode. Write the Python.

Each problem below shows AP pseudocode on the left. Write the equivalent Python in the editor (or on paper). When you're done, click "Show answer" to check your work — but try first.

AP Pseudocode → Python
Problem 1.1 Variables and display
AP Pseudocode
score ← 95 grade ← "A" DISPLAY score DISPLAY "Grade: " + grade
Your Python

Write this in the editor — then check your answer below.

Answer
score = 95 grade = "A" print(score) print("Grade: " + grade)
Problem 1.2 Conditional (IF/ELSE)
AP Pseudocode
temp ← INPUT() IF temp > 100 THEN DISPLAY "Too hot" ELSE DISPLAY "OK" ENDIF
Your Python

Hint: INPUT() with no prompt just means input(). You'll need float() or int() to compare to a number.

Answer
temp = float(input()) if temp > 100: print("Too hot") else: print("OK")
Problem 1.3 REPEAT n TIMES loop
AP Pseudocode
total ← 0 REPEAT 5 TIMES { num ← INPUT() total ← total + num } DISPLAY total
Your Python

REPEAT n TIMES maps to a for loop with range(). The curly braces {} mark the loop body — use indentation in Python.

Answer
total = 0 for i in range(5): num = float(input()) total = total + num print(total)
Problem 1.4 Lists and FOR EACH
AP Pseudocode
scores ← [85, 92, 78, 96] FOR EACH s IN scores { IF s ≥ 90 THEN DISPLAY s ENDIF }
Your Python

FOR EACH maps directly to Python's for loop. ≥ becomes >=.

Answer
scores = [85, 92, 78, 96] for s in scores: if s >= 90: print(s)
Problem 1.5 PROCEDURE (function)
AP Pseudocode
PROCEDURE double(n) { RETURN n * 2 } result ← double(7) DISPLAY result
Your Python

PROCEDURE becomes def. Everything else maps directly.

Answer
def double(n): return n * 2 result = double(7) print(result)
Part 2 — Python → AP pseudocode

Read the Python. Write the pseudocode.

This direction is harder — and more valuable. When you can translate Python into AP pseudocode, you understand what your program is actually doing beneath the syntax. Write your answers in the text boxes below.

Python → AP Pseudocode
AP pseudocode reminders

Assignment uses ← (not =). Output uses DISPLAY. Input uses INPUT(). Loops use REPEAT n TIMES { } or FOR EACH item IN list { }. Functions use PROCEDURE name(param) { RETURN value }. Lists are 1-indexed.

Problem 2.1 Input and arithmetic
Python (given)
hours = float(input("Hours worked: ")) rate = float(input("Hourly rate: ")) pay = hours * rate print("Total pay: " + str(pay))
Your AP pseudocode
Answer
hours ← INPUT() rate ← INPUT() pay ← hours * rate DISPLAY "Total pay: " + pay
Problem 2.2 While loop → REPEAT UNTIL
Python (given)
count = 1 while count <= 5: print(count) count = count + 1
Your AP pseudocode
Answer
count ← 1 REPEAT UNTIL count > 5 { DISPLAY count count ← count + 1 }
Problem 2.3 Function with return value
Python (given)
def is_passing(score): if score >= 60: return True else: return False result = is_passing(72) print(result)
Your AP pseudocode
Answer
PROCEDURE is_passing(score) { IF score ≥ 60 THEN RETURN true ELSE RETURN false ENDIF } result ← is_passing(72) DISPLAY result
The hardest part of this exercise

Problem 2.2 trips most students: while count <= 5 becomes REPEAT UNTIL count > 5. The condition flips because while runs as long as something is true, but REPEAT UNTIL runs until something becomes true. Same loop, opposite condition.

Why this exercise matters in April

AP exam MCQ questions regularly give you AP pseudocode and ask what it outputs, or give you a program description and ask which pseudocode correctly implements it. Students who have never practiced reading pseudocode spend valuable exam time decoding notation instead of thinking about the logic. You've now practiced both directions — that's an advantage.
