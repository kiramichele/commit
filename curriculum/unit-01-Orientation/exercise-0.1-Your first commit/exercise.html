Python strings 101

Everything inside the quotation marks is printed exactly as written — spaces, punctuation, capitalization included. If you put "hello" you get hello. If you put "Hello!" you get Hello!. The quotes themselves never appear in the output.

Your commit message

Writing a message that actually means something

After you run your code and it looks right, click Commit. You'll be asked to write a short message describing what you did. Here's the difference between a useful message and a useless one:

✕
"done"
Done what? In six months this tells you nothing.
✕
"asjdhfkajshdf"
This happens. It's fine once. Not a habit.
✓
"added my name, city, and learning goal"
Specific. Describes the change. Takes 5 extra seconds to write.
✓
"first working version of the intro program"
Also good — captures the milestone, not just the contents.
Why this habit matters

Your commit history is the record of how your thinking evolved on a project. On the AP Create PT, you'll write about your development process. Students with descriptive commit messages can look back at their history and reconstruct exactly what they tried and why. Students who typed "done" for every commit can't.

Common mistakes

Things to watch for

!
Forgetting the quotation marks

If you write print(Hello) instead of print("Hello"), Python looks for a variable named Hello — which doesn't exist. You'll get a NameError. Everything you want to print as text needs to be inside quotes.

!
Mismatched quotes

Opening with a double quote and closing with a single quote — or vice versa — will cause a SyntaxError. Python requires the quotes to match: "like this" or 'like this', not "like this'.

!
Running without reading the output

Always read what actually printed — not what you think you typed. Typos in strings are common and won't cause an error, they'll just print wrong. Check it before you commit.

Extension Want to go further? Add a fourth line. ▼

Add a fourth print() statement that uses a variable instead of putting the text directly inside the print call. Like this:

main.py — extension
8.
9# Store a value in a variable first
10favorite_thing = "ceramics"
11print(f"One thing I do outside school: {favorite_thing}")

The f"..." is called an f-string — the f stands for "formatted." Anything inside curly braces {} gets replaced by the value of that variable when the line runs. You'll use f-strings constantly this year.

Try changing the value of favorite_thing and running it again. Notice you only have to change it in one place — not in the print statement. That's the point of variables.
